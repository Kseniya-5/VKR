# Автоматическая генерация аннотаций для датасета DeepFashion-1

**Автор:** Балицкая Ксения

Данный модуль отвечает за автоматическую разметку (аннотирование) изображений одежды из датасета DeepFashion-1 с использованием локальной мультимодальной нейронной сети. 

Процесс полностью автоматизирован: скрипт очищает исходные данные от мусора, случайным образом выбирает заданное количество фотографий, копирует их в директорию для обучения и отправляет на анализ локальной модели компьютерного зрения для извлечения детальных fashion-атрибутов (цвет, фасон, материал, возрастная группа и т.д.)

## Аппаратное обеспечение

Для обеспечения безопасности данных и независимости от облачных квот, инференс (генерация аннотаций) производился на локальном сервере со следующими характеристиками:

* **Процессор (CPU):** Intel Core i7-13700K
* **Видеокарта (GPU):** NVIDIA GeForce RTX 4080 (16 GB VRAM)
* **Оперативная память:** 32 GB DDR5

***

##  Используемые модели и API

В качестве "разметчика" (асессора) используется легковесная мультимодальная модель от Meta:
* **Модель:** `llama3.2-vision:latest`
* **Оркестрация и API:** Модель запущена через систему **Ollama** и управляется веб-интерфейсом **Open WebUI**. 

Скрипт связывается с локальным сервером через OpenAI-совместимое API, предоставляемое Open WebUI по локальной сети, что обеспечивает высокую скорость обработки без задержек интернет-соединения.

## Подготовка исходных данных (DeepFashion)

Исходные изображения (более 440 000 фотографий) были загружены из открытого репозитория датасета **DeepFashion-1**:
[GitHub: carolinexull/DeepFashion-1](https://github.com/carolinexull/DeepFashion-1)

Для обучения и тестирования в рамках данного проекта из общей массы случайным образом (с фиксированным `seed=42`) отбирается **15 000 изображений**. Отобранные фотографии копируются в плоскую структуру в папку `data/images_for_training/` для дальнейшей работы загрузчиков данных (DataLoaders).

## Настройка окружения (Для самостоятельного запуска)

Для запуска скрипта `create_dataset.py` необходимо создать файл `.env` (в одной директории со скриптом) и указать в нем сетевые доступы к вашему серверу Open WebUI.

Пример содержимого файла `.env`:
```env
# IP-адрес компьютера, на котором запущен Open WebUI
API_URL="http://192.168.1.14:3000/api/chat/completions"

# JWT Токен или API ключ, сгенерированный в настройках Open WebUI
API_KEY="sk-ваш_секретный_ключ"
```

*Файл `config.py` автоматически подтягивает эти переменные для использования в скрипте.*

## Запуск скрипта

1. Убедитесь, что исходные картинки распакованы в папку `data/images`.
2. Активируйте виртуальное окружение и установите зависимости (`requests`, `pillow`, `tqdm`).
3. Запустите скрипт:

```bash
uv run create_dataset.py
```

### Описание процесса работы скрипта:
1. **Очистка мусора:** Удаление системных файлов (например, `Zone.Identifier`).
2. **Семплирование:** Отбор 15 000 фото и их безопасное копирование в `data/images_for_training/` с переименованием во избежание конфликтов.
3. **Анализ (Инференс):** Итеративная отправка изображений (в формате base64) в `llama3.2-vision`.
4. **Сохранение (Checkpointing):** Результаты записываются в файл `data/dataset_annotations.json`. Скрипт поддерживает возобновление работы с места обрыва в случае отключения света или перезагрузки сервера.
